# ⚖️ Локальный помощник по НПА в сфере информационной безопасности

Полностью локальная вопросно-ответная система (ВОС) на основе **RAG (Retrieval-Augmented Generation)** и **RAPTOR** для анализа нормативно-правовых актов РФ в области ИБ.  
Работает без интернета (после установки), все данные хранятся внутри папки.

## 🚀 Быстрый старт (для обычного пользователя)

### 1. Установите необходимое ПО (один раз)

Системе требуется **Visual C++ Build Tools** для сборки одного из компонентов. Это стандартная бесплатная утилита от Microsoft.

- Скачайте установщик: [Visual C++ Build Tools](https://visualstudio.microsoft.com/ru/visual-cpp-build-tools/) (выберите бесплатную версию)
- Запустите установку, выберите **«Разработка классических приложений на C++»** (это около 4 ГБ).  
  > Если места мало, можно установить только компонент **MSVC v143** и **Windows 10 SDK**.
- Перезагрузите компьютер.
- Установите Pyton(или убедитесь что установлена нужная версия) версии 3.12.*.
   -проверьте версию
	py -0
   - если нужной нет установите
	winget install -e --id Python.Python.3.12

### 2. Установите Ollama

- Перейдите в папку ollama и запустите OllamaSetup.exe.
- Извлеките файл `ollama.exe` и положите его в папку `ollama` внутри проекта (создайте её, если нет).

### 3. **Откройте командную строку (cmd) от имени пользователя** и выполните:

cd C:\local_law_qa

# Временно отключаем зеркало Яндекса (удаляем конфигурацию pip)
pip config unset global.index-url

# Устанавливаем Cython с официального PyPI
pip install cython

# Устанавливаем все зависимости с официального PyPI
pip install -r requirements.txt

# После установки можно вернуть зеркало (не обязательно)
pip config set global.index-url https://mirror.yandex.ru/pypi/simple/

### 4. Запустите систему

- Дважды кликните по файлу **`start.bat`**.
- Первый запуск может занять 5–10 минут – будет установка пакетов Python и загрузка моделей (около 5 ГБ). Интернет нужен только один раз.
- После запуска в браузере автоматически откроется адрес: [http://localhost:8501](http://localhost:8501)

## 📂 Как пользоваться

### Загрузка документов
- Скопируйте ваши PDF, DOCX или TXT файлы в папку `data/laws/`.
- Либо нажмите в боковой панели **«Добавить документы»** и выберите файлы.

### Индексация
- В боковой панели нажмите **«Переиндексировать всё»**.
- Система разобьёт документы на фрагменты и построит векторную базу. При первом включении RAPTOR также построит иерархическое дерево (это может занять дополнительное время).

### Задавайте вопросы
- Введите вопрос в поле чата (например: *«Какие требования к обработке персональных данных?»*).
- Для сложных концептуальных вопросов включите чекбокс **«Использовать RAPTOR»** в боковой панели.
- Ответ появится вместе со ссылками на источники.

## ⚙️ Настройки
- **Креативность ответов** – температура LLM (0.0 – строго по фактам, 1.0 – более свободные формулировки).
- **Количество фрагментов** – сколько кусков документов учитывать при поиске.

## 🧹 Очистка базы
- Кнопка **«Очистить базу»** удаляет все индексы (нужно будет переиндексировать документы заново).
- **«Очистить историю чата»** – удаляет только переписку.

## ❓ Частые проблемы

| Проблема | Решение |
|----------|---------|
| Ошибка `ModuleNotFoundError: No module named 'llama_index'` | Повторно запустите `start.bat`. Убедитесь, что интернет был активен во время установки. |
| Ошибка `Failed to build numpy` | Установите Visual C++ Build Tools (см. шаг 1). После установки перезапустите `start.bat`. |
| Ollama не запускается | Проверьте, что `ollama.exe` лежит в папке `ollama`. Запустите вручную `ollama\ollama.exe serve` и посмотрите на ошибки. |
| Вопрос не находит документы | Убедитесь, что вы нажали **«Переиндексировать всё»**. Проверьте, что документы лежат в `data/laws/`. |
